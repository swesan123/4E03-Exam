# 4E03-Exam

Tooling to extract sample questions from PDFs, emit LaTeX banks, and generate practice exams with usage tracking.

## Prerequisites

- **Python 3.10+**
- **`pdftotext`** (Poppler), e.g. `sudo apt install poppler-utils`
- **LaTeX**: `latexmk` + **LuaLaTeX** (recommended; used automatically when `lualatex` is on `PATH`), e.g. `sudo apt install texlive-luatex latexmk`. Falls back to **pdfLaTeX** with a limited Unicode map if LuaLaTeX is not installed.
- Python deps: `pip install -r requirements.txt` (PyYAML for optional tag overrides)

## Layout

| Path | Purpose |
|------|---------|
| [`sample questions/`](sample%20questions/) | Source PDFs (`exam_sample_questions.pdf` ‚Üí category **newer**; filenames containing `test1` / `test2` ‚Üí those categories) |
| [`data/questions.json`](data/questions.json) | Extracted question bank (regenerated by extract) |
| [`data/question_tags.yaml`](data/question_tags.yaml) | Optional per-id overrides: `test1`, `test2`, or `newer` |
| [`data/usage.json`](data/usage.json) | How often each question appeared in generated exams (for coverage) |
| [`build/latex/`](build/latex/) | LaTeX fragments: `exam_bank.tex`, `test1_bank.tex`, `test2_bank.tex` |
| [`generated_exams/`](generated_exams/) | Timestamped `practice_exam_*.pdf` only |
| [`generated_exams_work/`](generated_exams_work/) | Working files (`.tex`, `.aux`, `.log`, etc.) |

## Workflow

**From the repo root** (recommended): use the launcher scripts‚Äîthey delegate to `scripts/` and fix `PYTHONPATH` for `exam_toolkit`.

```bash
# 1) Extract numbered questions (left-margin "N. " only; strips page footers)
python3 extract_questions.py --sample-dir "sample questions"

# 2) Build LaTeX banks (applies question_tags.yaml overrides)
python3 build_latex.py

# 3) Generate a practice exam: writes .tex and runs latexmk (LuaLaTeX by default) to produce a PDF
python3 generate_practice_exam.py --seed 1
```

Use `python3 generate_practice_exam.py --no-compile` if you only want the `.tex` file (no PDF build). Use `--latex-engine pdflatex` or `xelatex` to force an engine.
Use `python3 generate_practice_exam.py --clear-usage-only` to reset pick history, or `--reset-usage` to clear then generate.

Or run the implementations directly from `scripts/`:

```bash
cd scripts
python3 extract_questions.py --sample-dir "../sample questions"
python3 build_latex.py
python3 generate_practice_exam.py --seed 1
```

### Extraction rules

- Question boundaries: lines matching `^ {0,2}(\d+)\. ` (0‚Äì2 leading spaces). This avoids false splits on indented lines such as `23. The average response time‚Ä¶` inside a T/F list.
- **True/False** items are detected from the first line of the stem (`True or False`, `True/False`, `Answer True or False`).
- Page artifacts (`-- N of M --`, isolated 1‚Äì2 digit lines) are removed after `pdftotext -layout`.

### Practice exam generator

- Picks **eight** non‚ÄìT/F questions with **stratified** quotas across `test1` / `test2` / `newer` (as evenly as possible given availability). If a category has no calculation questions, a warning is printed and picks are redistributed.
- Picks **one** T/F question, preferring at least `--min-tf-subparts` lettered sub-parts (default 10); falls back to the richest T/F available.
- Weighting: each question‚Äôs weight is \(\propto 1/(1+\text{usage count})\) to spread coverage over repeated runs.
- **`--dry-run`**: print selections without writing TeX or updating `usage.json`.
- **`--preserve-layout`** (default on): keeps line-based formatting from the source PDF using `Verbatim` blocks; disable with `--no-preserve-layout`.
- **`--clear-usage-only` / `--reset-usage`**: clear question-pick history in `data/usage.json`.

Example flags:

```bash
python3 generate_practice_exam.py --marks 10,10,10,10,10,10,15,5 --tf-marks 10 --with-answers
```

### Compiling LaTeX

- **Practice exams** use **LuaLaTeX** when available (best match for Unicode taken from PDFs). Otherwise they use **pdfLaTeX** with `newunicodechar` mappings in [`scripts/exam_toolkit/tex_preamble.py`](scripts/exam_toolkit/tex_preamble.py). Private-use (matrix) glyphs are stripped in [`scripts/exam_toolkit/latex_escape.py`](scripts/exam_toolkit/latex_escape.py).
- **LaTeX Workshop**: default recipe is **latexmk (lualatex)** in [`.vscode/settings.json`](.vscode/settings.json).
- Question banks are plain UTF-8; auto-escaping is minimal‚Äîspot-check the PDF for math-heavy lines.

## Optional tag overrides

Edit [`data/question_tags.yaml`](data/question_tags.yaml) to force a question id into another bucket (keys must match `id` in `questions.json`):

```yaml
exam_sample_questions_q5: test1
```
