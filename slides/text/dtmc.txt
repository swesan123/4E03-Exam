Discrete Time Markov Chains

--- PAGE 2 ---
Stochastic Models
May want to capture how a system evolves, using dependent
random variables, described by some dynamics. A very simple
model that sees wide application is a Discrete Time Markov
Chain. They have been used for the following (this is far from
exhaustive):
user web page navigation, cache contents and performance,
speech recognition, machine learning (reinforcement
learning), algorithmic music composition, sports

--- PAGE 3 ---
Introductory Example
Consider the following (very simplified) scenario:
• three web pages that link to each other A, B, C
• page A has links to B and C
• page B has links to C
• page C has links to A and B
• when a user leaves A, 1/3 of the time a link to B is selected,
otherwise a link to C is selected
• B always links to C
• C links to A or B with equal probability
• Choice of next page is completely independent of the past (only
depends on current page)
Q: What proportion of visits are made to each page?

--- PAGE 4 ---
Discrete-time systems
• this is a discrete-time system
• the nth “time point” corresponds to the nth page visited
• natural for some time settings: a continuous-time framework
may be more suitable for others

--- PAGE 29 ---
Example
Consider a multiprocessor system with two processors and two
memory modules (MMs). At each time point, requests for an MM
are made in the following manner.
1. If both processors are accessing different MMs, at the next
time point each processor makes independent requests for the
memory modules: with probability 0.4, a request is made for
MM1, otherwise the request is made for MM2.
2. If both processors are accessing the same MM, only one
processor makes a new request at the next time step (the
other maintains its request for the same MM). The request
probabilities are the same as in the first case.
Calculate the steady-state probability that the processors are
accessing different memory modules.
