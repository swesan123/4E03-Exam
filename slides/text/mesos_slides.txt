--- PAGE 9 ---
Mesos Scheduler
• Mesos scheduler sends resource offers to frameworks
• Frameworks select which offers to accept and which tasks to run
• Unit of allocation: resource offer
• Vector of available resources on a node

--- PAGE 10 ---
Mesos Scheduler: Example
1. Node 1 sends message to Mesos scheduler: <node 1, 4cpu, 4GB>
(nodes periodically send status update to Mesos scheduler)
2. Mesos scheduler sends resource offer to a framework, for example
Framework 1: <node 1, 4cpu, 4GB>
3. Framework 1 responds with either rejecting offer or accepting with
a task list to be run: <task1, node1, 2cpu, 1GB>; <task2, node1,
1cpu, 1GB> (these choices made according to framework
scheduler’s rules)
4. Resulting tasks sent to nodes

--- PAGE 12 ---
Single Resource – Fair Sharing
1. Suppose that 𝑛 users want to shar a single resource, say CPU
• Allocate each 1/𝑛 of the shared resource

2. If a user wants less than its fair share, then generalized by max-min
fairness:
• Suppose there are three users, under fair sharing each gets 33% of the
resource
• If one user only wants 20%, then that user gets 20%; the remaining users get
40%

3. Can generalize with weights to reflect importance.
• For example, two users, one with weight 2, the other with weight 1: first user
gets 66%, the second user gets 33%

--- PAGE 13 ---
Properties of Max-Min Fairness
• Share guarantee
• Each user can get at least 1/𝑛 of the resource
• Will get less if demand is less

• Strategy proof
• Users are not better off by asking for more than they need
• Users have no reason to lie

• When is this not enough?
• Multiple resources!

--- PAGE 15 ---
A Natural Policy
• Asset fairness: give weights to resources (e.g. 1 CPU = 1 GB) and
equalize total value given to each user
• Total resources: 28 CPU and 56 GB RAM (so here, a natural choice is 1
CPU = 2 GB)
• User 1 wants <1 CPU, 2 GB> per task
• User 2 wants <1 CPU, 4 GB> per task

• Asset fairness yields the following problem:
max 𝑥, 𝑦
𝑥 + 𝑦 ≤ 28
2𝑥 + 4𝑦 ≤ 56
2𝑥 = 3𝑦
• Result gives 𝑥 = 12 and 𝑦 = 8, so User 1 gets 43% of the CPUs and
43% of the RAM; User 2 gets 28% of the CPUs and 57% of the RAM

--- PAGE 16 ---
A Natural Policy
• Problem: User 1 gets less than 50% of both CPU and RAM, so would
be better off in a separate cluster with half the resources

--- PAGE 17 ---
Dominant Resource Fairness (DRF)
• Dominant resource of a user: the resource that user has the biggest
share of:
• Total resources: <8 CPU, 5 GB>
• User 1 allocation: <2 CPU, 1 GB>: 2/8=25% CPU and 1/5=20% RAM
• Dominant resource of User 1 is CPU (25% > 20%)

• Dominant share of a user: the fraction of the dominant resource they
are allocated
• User 1 dominant share is 25%

--- PAGE 18 ---
Dominant Resource Fairness (DRF)
• Apply max-min fairness to dominant shares: give every user an equal
share of their dominant resource
• Example
• total resources <9 CPU, 18GB>
• User 1 tasks need <1 CPU, 4GB>; dominant resource RAM (1/9 < 4/18)
• User 2 tasks need <3 CPU, 1GB>; dominant resource CPU (3/9 > 1/18)

• Solve the following problem
max 𝑥, 𝑦
𝑥 + 3𝑦 ≤ 9
4𝑥 + 𝑦 ≤ 18
4𝑥 3𝑦
=
18
9
• Gives 𝑥 = 3 (33% CPU, 66% RAM) and 𝑦 = 2 (66% CPU, 16% RAM)
